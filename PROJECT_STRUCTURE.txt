# KEY FILES CONTENT SUMMARY
# ==========================

1. MAIN ENTRY POINT (main.py)
   - Parses command-line arguments (--target_dir)
   - Validates target directory exists
   - Initializes Orchestrator
   - Calls orchestrator.refactor_directory()
   - Logs startup to experiment_data.json

2. ORCHESTRATOR (src/orchestrator.py)
   - Discovers all Python files in target directory
   - For each file, runs max 10 iterations of:
     a) Auditor.analyze() → get issues list
     b) Fixer.fix() → apply fixes → refactored code
     c) Judge.validate() → check quality → verdict
     d) validate_syntax() → ensure refactored code is valid
     e) run_tests() → execute pytest
   - Stops when:
     - Auditor finds no issues
     - Judge gives APPROVED verdict and tests pass
     - Max iterations reached
   - Stores results in self.results dict

3. AUDITOR AGENT (src/agents/auditor.py)
   - Extends BaseAgent
   - analyze(file_path, code) → Dict with issues
   - Uses auditor_prompt.txt as system instructions
   - Returns JSON with:
     - file
     - total_issues
     - issues[] (with severity, type, description, line, snippet, recommendation)
     - overall_quality_score (0-100)

4. FIXER AGENT (src/agents/fixer.py)
   - Extends BaseAgent
   - fix(file_path, code, issues) → Dict with refactored code
   - Uses fixer_prompt.txt as system instructions
   - Returns JSON with:
     - file
     - refactored_code (complete Python code)
     - changes[] (detailed list of each change)
     - summary (overall summary of changes)

5. JUDGE AGENT (src/agents/judge.py)
   - Extends BaseAgent
   - validate(file_path, original_code, refactored_code, changes_summary) → Dict
   - Uses judge_prompt.txt as system instructions
   - Returns JSON with:
     - file
     - verdict (APPROVED | NEEDS_REVISION | REJECTED)
     - quality_improvement_score (-100 to +100)
     - issues_fixed (number)
     - new_issues_introduced (number)
     - feedback (detailed explanation)
     - suggestions[] (if NEEDS_REVISION)

6. BASE AGENT (src/agents/base_agent.py)
   - Abstract base class for all agents
   - Features:
     * call_gemini_api(user_message) → API response
     * Response caching by content hash → logs/api_response_cache.json
     * Request throttling (2s between calls)
     * Retry logic with exponential backoff (5s → 10s → 20s → 60s)
     * Quota exceeded (429) detection and handling
     * JSON parsing with fallback
     * Structured logging via log_experiment()
   - All agents inherit from BaseAgent

7. LOGGING SYSTEM (src/utils/logger.py)
   - Structured JSON logging
   - ActionType enum (ANALYSIS, GENERATION, DEBUG, FIX)
   - log_experiment() function
   - Validates input (requires input_prompt and output_response)
   - Appends to logs/experiment_data.json
   - Each entry has: id, timestamp, agent, model, action, details, status

8. TOOLS MODULE (src/tools/__init__.py)
   - discover_python_files(target_dir) → List[str] of file paths
   - read_file(file_path) → str
   - write_file(file_path, content) → None
   - validate_syntax(code, file_path) → Dict[valid, errors]
   - run_tests(test_dir) → Dict[passed, stdout, stderr]

9. SYSTEM PROMPTS (src/prompts/*.txt)
   All are plain text files with detailed instructions:
   
   a) auditor_prompt.txt
      - Instructions for analyzing Python code
      - Severity levels: LOW, MEDIUM, HIGH
      - Issue types: Code Quality, Performance, Security, Maintainability, Type Safety
      - Expected JSON output structure
   
   b) fixer_prompt.txt
      - Instructions for refactoring code
      - Guidelines: preserve functionality, add type hints, add docstrings
      - Expected JSON output with refactored_code, changes, summary
   
   c) judge_prompt.txt
      - Instructions for validating refactored code
      - Evaluation criteria: functionality, issues fixed, new issues, readability
      - Verdicts: APPROVED, NEEDS_REVISION, REJECTED

10. ENVIRONMENT (.env)
    - GOOGLE_API_KEY=<your-api-key>
    - Loaded by load_dotenv() in base_agent.py
    - Template provided in .env.example

         │

         │    └─ run_tests() → pytest
         │       ├─ Tests pass → SUCCESS (exit loop)
         │       └─ Tests fail → Continue iteration

         └─ Next iteration (if < max_iterations)


# AGENT COMMUNICATION PROTOCOL
# ==============================

All agents use Google Gemini API via google.genai Client:

REQUEST:
├─ Model: "gemma-3-4b-it" (or "gemini-2.5-flash")
├─ System Prompt: Loaded from src/prompts/<agent>_prompt.txt
└─ User Message: File path + code + specific task

RESPONSE:
├─ Expected: JSON object
├─ Fallback: Extract JSON from text response
└─ Error: Return error structure

CACHING:
├─ Key: MD5(model + system_prompt + user_message)
├─ Storage: logs/api_response_cache.json
└─ Hit rate: Reduces API calls significantly

RATE LIMITING:
├─ Throttle: 2 seconds between API calls
├─ Retry: Exponential backoff on 429 errors
└─ Max backoff: 60 seconds
